What is a GitHub work flow?
Everything starts with a master branch, when you try to make some changes it will be save in a local branch which is called feature. When you are done with your changes or ready to share it to the master branch you will need to open a pull request to be further assess and tested. If all is satisfied, it will then merge to the master branch.

Branch, Pull, Merge Primer
What is Master?
Master Branch is the main active files. This branch is mostly used for production or is used actively by the public with little to minor errors.
Why create a Branch?
Branch can help prevent mistakes or errors in the codes. While in branch, the user can experiment with whatever he/she wants to do with those files without affecting the master branch.

GitHub Fork & Clone
What are some examples of when you would fork?
I would fork a project if I had some idea to improve the existing codes and either create a pull request or use it on my own as long as it's an open source.
Write your own step-by-step fork and clone instructions. Describe what you did.

For Forking: Navigate to the repo you want to copy, then on the upper middle left side - click on Fork. A pop up will appear on where to fork that repo (Your repo should be there once you have logged in). Check your repo and there it is!

For Cloning: Navigate to your collection of repo that you want to clone. Click on Clone or download button (color Green). Copy the link. Open your terminal, type in: git clone (then paste here the link you copied earlier) (optional type in the folder name that you want) then press enter. cd to the folder and that's it!
